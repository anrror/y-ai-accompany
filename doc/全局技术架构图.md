## 2. 对话管线数据流图（12-Step Pipeline with LLM 驾驭工程）

```mermaid
sequenceDiagram
    actor User as 用户
    participant GW as API Gateway
    participant ORCH as Orchestrator
    participant GD as Guard Service
    participant EMO as Emotion Detector
    participant SES as Session Manager
    participant REG as Agent Registry
    participant MEM as Memory Engine
    participant PER as Personality Engine
    participant CTX as Context Builder
    participant DRV as LLM Driver<br/>驾驭工程层
    participant LLM as LLM (Qwen3.6-27B)
    participant EMO_MODEL as HumanOmni-Emotion<br/>专用情感模型
    participant DIST as Async Workers
    participant DB as PostgreSQL + Redis

    User->>GW: POST /v1/chat/completions<br/>{user_id, agent_id, message}
    GW->>ORCH: Forward authenticated request

    Note over ORCH: Step 1: Guard (Input)
    ORCH->>GD: Check user message
    GD->>GD: Layer 1-2: Pre-filter + Classify
    GD-->>ORCH: {safe: true}

    Note over ORCH: Step 2: Emotion Detection
    ORCH->>EMO: Detect emotion(message)
    EMO->>EMO: Level 1: Emoji match (fast path)
    EMO->>EMO: Level 2: Keyword match (fast path)
    EMO->>EMO_MODEL: Level 3: HumanOmni-Emotion
    EMO_MODEL-->>EMO: {emotion: "sadness", VAD: [0.20, 0.35, 0.30]}
    EMO-->>ORCH: {emotion: "sadness", intensity: 0.7, valence: -0.5}

    Note over ORCH: Step 3: Session Load
    ORCH->>SES: GetOrCreate(user_id, agent_id)
    SES->>DB: Redis HGETALL session state
    DB-->>SES: {status: active, turn: 42, intimacy: 65}
    SES-->>ORCH: Session state + renew TTL

    Note over ORCH: Step 4: Load Agent Config
    ORCH->>REG: Get(agent_id)
    REG->>DB: PostgreSQL SELECT agent_config
    DB-->>REG: AgentConfig {identity, personality, capabilities, memory_config}
    REG-->>ORCH: AgentConfig

    Note over ORCH: Step 5: Memory Retrieval
    ORCH->>MEM: Retrieve(user_id, agent_id, message, top_k=5)
    MEM->>DB: pgvector HNSW search (cosine)
    DB-->>MEM: Top 15 candidates
    MEM->>MEM: Reranker re-rank
    MEM-->>ORCH: Top 5 memories with scores

    Note over ORCH: Step 6: Personality Rules
    ORCH->>PER: BuildRules(personality)
    PER-->>ORCH: "You are deeply empathetic..."

    Note over ORCH: Step 7: Build System Prompt
    ORCH->>CTX: Build(agent, personality_rules, memories, emotion, intimacy)
    CTX-->>ORCH: Complete System Prompt

    Note over ORCH: Step 8: LLM 驾驭工程 Orchestration
    ORCH->>DRV: ChatStream(messages, emotion, personality)
    DRV->>DRV: ParameterTuner: VAD→tune temperature<br/>OCEAN→叠加人格调优
    DRV->>DRV: TokenBudgetManager: 分配上下文窗口<br/>System 25% / History 45%
    DRV->>DRV: RetryStrategy: 指数退避<br/>熔断器检查
    DRV->>DRV: MetricsCollector: 记录调用指标

    Note over ORCH: Step 9: LLM Streaming
    DRV->>LLM: ChatStream(messages, tuned_config)
    LLM-->>DRV: SSE stream: "我" "在这里" "听着呢" "..."
    DRV-->>ORCH: SSE stream (with metrics)

    Note over ORCH: Step 10: Guard (Output)
    ORCH->>GD: Check full reply
    GD->>GD: Layer 4: Output classifier
    GD-->>ORCH: {safe: true}

    Note over ORCH: Step 11: Persist
    ORCH->>DB: Save user + assistant messages to working memory (Redis)

    Note over ORCH: Step 12: Stream Response
    ORCH-->>User: SSE Stream: "我在这里听着呢..."

    Note over ORCH,DIST: Async Post-Processing
    ORCH-)DIST: Trigger post_process(user_id, agent_id)
    DIST->>LLM: Extract memories (summary, facts, topics)
    LLM-->>DIST: {summary: "...", facts: ["..."], topics: ["..."]}
    DIST->>DB: Save short-term memories (pgvector)
    DIST->>DB: Save long-term facts (if frequency >= 3)
    DIST->>PER: Evolve personality (decay 0.97)
    PER->>DB: Update personality params
```

## 4. 内存记忆管线架构图

```mermaid
graph LR
    subgraph Input["每轮对话输入"]

    end

    end

    subgraph Scratch["Scratchpad"]
        ID[identity<br/>用户身份信息]
        PREF[preference<br/>用户偏好]
        GOAL[goal<br/>用户目标]
        EVT[event<br/>用户事件]
        REL[relationship<br/>关系信息]
        FACT[fact<br/>事实信息]
    end

    subgraph Distill["记忆蒸馏 (LLM)"]
        EXTRACT[LLM 结构化提取<br/>summary + facts + topics + preferences]
        DEDUP[去重 + 矛盾检测]
    end

    subgraph ST["短期记忆 (pgvector)"]
        ST_VEC[蒸馏摘要<br/>4096d 向量<br/>TTL: 90天]
    end

    subgraph LT["长期记忆 (pgvector)"]
        LT_VEC[固化事实<br/>frequency ≥ 3<br/>永久存储]
    end

    subgraph Retrieval["记忆检索"]
        QUERY[用户当前消息]
        Q_EMB[Query Embedding<br/>4096d]
        HNSW[pgvector HNSW<br/>向量检索 top_k×3]
        RERANK[Reranker 精排]
        RESULT[Top K 记忆]
    end

    MSG --> TURNS
    TURNS -->|每 50 条触发| EXTRACT
    Scratch -.->|LLM 管理 CRUD| EXTRACT
    EXTRACT --> DEDUP
    DEDUP -->|summary| ST_VEC
    DEDUP -->|高频事实 ≥ 3| LT_VEC

    QUERY --> Q_EMB
    Q_EMB --> HNSW
    ST_VEC --> HNSW
    LT_VEC --> HNSW
    HNSW --> RERANK
    RERANK --> RESULT
```

## 附：图例说明

| 符号 | 含义 |
|------|------|
| 实线箭头 `→` | 同步调用 / 数据流 |
| 虚线箭头 `-.->` | 异步调用 / 依赖关系 |
| 方框 `[...]` | 处理模块 |
| 圆柱 `[(...)]` | 数据库 / 持久存储 |
| 圆角框 `(...)` | 外部实体 |
| 六边形 `{{...}}` | 判定 / 条件分支 |
