### 1.3 注销与清理流程

```
注销触发: POST /v1/agents/{agent_id}/unregister

平台执行:
  1. Registry: 标记 status = "inactive"，从内存映射中移除
  2. Memory Engine: 
     ├── 删除 Redis 中该 Agent 的所有会话状态
     ├── 删除 Redis 中该 Agent 的所有工作记忆
     └── 删除 PostgreSQL 中该 Agent 的所有记忆数据
         (DELETE FROM agent_memories WHERE agent_id = $1)
  3. Session Manager: 清理所有与该 Agent 关联的会话缓存
  4. 返回: { status: "unregistered" }

注意: 注销操作删除该 Agent 的跨所有用户记忆数据。
      Agent 的配置记录保留在 PostgreSQL 中 (status = inactive)。
```

## 第二部分：Memory Engine 详细设计

### 2.1 架构位置

```
对话编排器 ──→ Memory Engine (平台服务 MaaS)
                    ↑
                    │ 检索/存储/蒸馏/管理
                    │
               ┌────┴────────────┐
               │                  │
          PostgreSQL + pgvector   Redis
          ────────────────────    ─────
          • 短期记忆向量           • 工作记忆 (List)
          • 长期记忆向量           • 会话状态 (Hash)
          • Scratchpad       • 速率限制 (ZSet)
```

### 2.2 检索实现（伪代码）

```
function retrieve_memories(user_id, agent_id, query, top_k=5):
    # 1. 向量化查询
    query_embedding = embedding_model.embed(query)  # 4096d

    # 2. pgvector HNSW 检索
    candidates = db.query("""
        SELECT id, content, memory_type, importance, 
               1 - (embedding <=> $1::vector(4096)) AS score
        FROM agent_memories
        WHERE user_id = $2 AND agent_id = $3
        ORDER BY embedding <=> $1::vector(4096)
        LIMIT $4
    """, query_embedding, user_id, agent_id, top_k * 3)

    # 3. Reranker 精排
    reranked = reranker.rerank(
        query=query,
        documents=[c.content for c in candidates],
        top_n=top_k
    )

    # 4. 返回结果
    return [
        {
            "content": candidates[r.index].content,
            "memory_type": candidates[r.index].memory_type,
            "score": r.relevance_score,
            "importance": candidates[r.index].importance
        }
        for r in reranked
    ]
```

### 2.3 HNSW 向量索引参数

```sql
-- 创建 HNSW 索引
CREATE INDEX IF NOT EXISTS idx_memories_embedding_hnsw 
ON agent_memories 
USING hnsw (embedding vector_cosine_ops) 
WITH (m = 16, ef_construction = 128);

-- 查询时设置 ef_search
SET hnsw.ef_search = 40;  -- 低延迟模式
-- 或
SET hnsw.ef_search = 80;  -- 高召回模式

-- 查询示例
SELECT id, content, memory_type, importance,
       1 - (embedding <=> $1::vector(4096)) AS cosine_similarity
FROM agent_memories
WHERE user_id = $2 
  AND agent_id = $3
  AND (memory_type = 'short_term' OR memory_type = 'long_term')
ORDER BY embedding <=> $1::vector(4096)
LIMIT 15;
```

### 2.4 记忆蒸馏管线（详细）

```
触发条件: 工作记忆消息数 ≥ 50

蒸馏流程:
1. 从 Redis 读取工作记忆 (最近 50 条消息)
2. 格式化为对话文本
3. LLM 调用 (非流式, temperature=0.1, max_tokens=200)
   
   Prompt:
   "Extract worth-remembering info from this conversation. 
    Return JSON only:
    {
      \"summary\": \"简要总结 (1-2句)\",
      \"topics\": [\"话题1\", \"话题2\"],
      \"emotion\": \"joy|sadness|anger|fear|surprise|neutral\",
      \"facts\": [\"用户提到的事实1\", \"事实2\"],
      \"preferences\": [\"用户偏好1\"]
    }
    
    Conversation:
    {conversation_text}"

4. 解析 JSON 结果
5. 存入短期记忆: 
   ├── content = summary
   ├── topics = extracted topics
   ├── emotion_label = detected emotion
   ├── embedding = embed(summary)  # 4096d
   └── memory_type = "short_term"

6. 高频事实固化:
   foreach fact in extracted.facts:
       similar = search_similar_facts(user_id, agent_id, fact)
       if count(similar) >= 2:  # 第 3 次提及
           store_long_term(fact, importance=0.8, is_permanent=false)
       elif count(similar) >= 5:  # 第 6 次提及
           mark_permanent(fact, is_permanent=true)

7. 清理工作记忆: 
   trim_working(user_id, agent_id, max_turns=15)
   # 保留最近 15 轮 (30 条消息)，删除已蒸馏的更早消息
```

### 2.5 Scratchpad 设计（未来增强）

```
Scratchpad 是介于工作记忆和短期记忆之间的 LLM 可管理结构化存储。

条目类型:
  identity:     { content, confidence: 0.9, last_mentioned: "2026-07-05" }
  preference:   { content, confidence: 0.8, last_mentioned: "2026-07-06" }
  goal:         { content, confidence: 0.7, last_mentioned: "2026-07-04" }
  event:        { content, confidence: 0.6, last_mentioned: "2026-07-07" }
  relationship: { content, confidence: 0.5, last_mentioned: "2026-07-03" }
  fact:         { content, confidence: 0.9, last_mentioned: "2026-07-07" }

管理操作 (LLM 通过 Function Calling 执行):
  CREATE:   "用户说他的名字是张三" → identity 条目
  UPDATE:   "用户改口说喜欢林俊杰而不是周杰伦" → 更新 preference 条目
  DELETE:   "用户说之前的过敏信息是错的" → 删除 fact 条目
  SEARCH:   "关于工作的话题" → 返回相关条目

容量: 最多 200 条。达到上限时按 (importance × recency) 淘汰最不重要的条目。

注意: Scratchpad 是增强功能，当前平台使用工作记忆 + 短期记忆 + 长期记忆即可满足 MVP 需求。
```

## 第三部分：对话管线详细设计

### 3.1 管线伪代码

```
async function chat_completion(user_id, agent_id, messages, stream=true):

    # Step 1: 安全审核 (用户侧)
    guard_result = guard.check(messages[-1].content)
    if not guard_result.safe:
        return error("message rejected by guard")

    # Step 2: 情绪感知
    emotion = emotion_detector.detect(messages[-1].content)
    # 返回: { emotion, intensity, valence }
    
    # Step 3: 会话管理
    session = session_manager.get_or_create(user_id, agent_id)
    if session.daily_usage_minutes >= 120:
        append_reminder("你已经连续使用 2 小时了，休息一下吧。")
    if session.is_minor and session.daily_usage_minutes >= 60:
        append_reminder("小朋友，今天的陪伴时间到了哦。明天见！")
    
    # Step 4: 加载 Agent 配置
    agent = registry.get(agent_id)
    if not agent:
        return error("agent not found")

    # Step 5: 检索记忆
    memories = memory_engine.retrieve(
        user_id, agent_id, 
        messages[-1].content, 
        top_k=agent.memory_config.retrieval_top_k
    )

    # Step 6: 生成性格规则
    personality_rules = personality_engine.build_rules(agent.personality)

    # Step 7: 构建 System Prompt
    system_prompt = context_builder.build({
        "agent_name": agent.identity.name,
        "persona": agent.identity.persona,
        "personality_rules": personality_rules,
        "speaking_style": agent.identity.speaking_style,
        "memories": format_memories(memories),
        "intimacy": session.intimacy_level,
        "emotion": emotion.emotion,
        "mode": select_conversation_mode(emotion)
    })

    # Step 8: 组装完整消息
    chat_messages = [
        {"role": "system", "content": system_prompt}
    ]
    working = memory_engine.get_working(user_id, agent_id)
    chat_messages.extend(working[-10:])  # 最近 10 条
    chat_messages.append(messages[-1])

    # Step 9: 用户消息入工作记忆
    memory_engine.add_working(user_id, agent_id, "user", messages[-1].content)

    # Step 10: LLM 驾驭工程 (Orchestration Layer)
    # 10a: 构建情绪上下文用于参数调优
    emotion_ctx = build_emotion_context(emotion)
    
    # 10b: Token 预算管理 (按比例分配上下文窗口)
    budget = token_budget_manager.budget(
        system_prompt=system_prompt,
        history=working[-10:],
        user_message=messages[-1].content
    )
    if budget.truncated:
        chat_messages = truncate_history(chat_messages, budget.used_history)
    
    # 10c: LLM 流式推理 (带重试和熔断)
    llm_config = agent.capabilities.llm_config
    tuned_temp = parameter_tuner.tune(
        base_temp=llm_config.temperature,
        emotion=emotion_ctx,
        personality=agent.personality
    )
    metrics_collector.record_start()
    
    full_reply = ""
    stream = llm_driver.chat_stream(
        model=llm_config.model,
        messages=chat_messages,
        temperature=tuned_temp,
        max_tokens=min(llm_config.max_tokens, budget.reserve),
        retry_strategy="exponential_backoff(max=3)",
        circuit_breaker="threshold=5, half_open_timeout=30s"
    )
    
    for chunk in stream:
        full_reply += chunk.content
        yield chunk  # SSE 输出

    # Step 11: 回复审核 + 持久化 + 异步后处理
    guard_result = guard.check(full_reply)
    if not guard_result.safe:
        yield "[内容已被安全策略过滤]"
        return

    memory_engine.add_working(user_id, agent_id, "assistant", full_reply)
    memory_engine.trim_working(user_id, agent_id, agent.memory_config.working_memory_turns)
    
    # 异步后处理
    spawn post_process(user_id, agent_id, messages[-1].content, full_reply, emotion, agent)


async function post_process(user_id, agent_id, user_msg, agent_reply, emotion, agent):
    # 记忆蒸馏
    if memory_engine.working_count(user_id, agent_id) >= 50:
        memory_engine.distill(user_id, agent_id)

    # 记忆提取 (每轮)
    extracted = memory_engine.extract_memories(user_msg, agent_reply)
    if extracted.summary:
        memory_engine.save_memory(user_id, agent_id, {
            content: extracted.summary,
            topics: extracted.topics,
            emotion_label: extracted.emotion,
            memory_type: "short_term",
            importance: 0.6
        })
    for fact in extracted.facts:
        memory_engine.save_memory(user_id, agent_id, {
            content: fact,
            memory_type: "long_term",
            importance: 0.8
        })

    # 性格演化
    personality_engine.evolve(agent.personality, {
        sentiment_valence: emotion.valence,
        topic_novelty: 0.5,
        user_engagement: len(user_msg) > 10 ? 0.6 : 0.3
    })

    # 轨迹审计 (每 20 轮)
    if session.turn_count % 20 == 0:
        guard.audit_trajectory(user_id, agent_id)
```

### 3.2 对话模式选择逻辑

```
function select_conversation_mode(emotion, message, session):
    
    # 危机模式 > 恢复模式 (最高优先级)
    if emotion.valence < -0.7 or contains_crisis_keywords(message):
        return "recovery"
    
    # 恢复模式 (次高优先级)
    if emotion.valence < -0.3:
        return "recovery"
    
    # 决策模式 (关键词匹配)
    if contains_decision_keywords(message):
        return "decision"
    
    # 反思模式 (深度对话)
    if contains_reflection_keywords(message) and session.turn_count > 10:
        return "reflection"
    
    # 成长模式 (探索性对话)
    if contains_growth_keywords(message):
        return "growth"
    
    # 默认陪伴模式
    return "companion"


危机关键词: ["不想活", "自杀", "结束一切", "死", "没有意义", "消失"]
决定关键词: ["怎么办", "选哪个", "要不要", "该不该", "帮我决定"]
反思关键词: ["为什么我", "我是什么样的人", "我是不是", "我的问题"]
成长关键词: ["怎么学", "如何提高", "想了解", "教我", "解释一下"]
```

## 第四部分：性格引擎详细设计

### 4.1 规则生成（纯函数，无状态）

```
function build_rules(personality):
    rules = []
    
    if personality.openness >= 0.65:
        rules.append("你好奇心强，喜欢探索新话题。鼓励用户尝试新事物。")
    elif personality.openness <= 0.35:
        rules.append("你偏好熟悉、舒适的话题和可预测的对话模式。")
    
    if personality.conscientiousness >= 0.65:
        rules.append("你说话有条理、逻辑清晰。提供明确的建议和计划。")
    elif personality.conscientiousness <= 0.35:
        rules.append("你随性随意，话题自然转换，不拘泥于结构。")
    
    if personality.extraversion >= 0.65:
        rules.append("你热情主动，经常发起话题，善于活跃气氛。")
    elif personality.extraversion <= 0.35:
        rules.append("你安静内敛，更多倾听而非主导对话。")
    
    if personality.agreeableness >= 0.65:
        rules.append("你深度共情，总是优先考虑对方的感受。")
    elif personality.agreeableness <= 0.35:
        rules.append("你直率坦诚，偶尔会温和地挑战用户的观点。")
    
    if personality.neuroticism >= 0.65:
        rules.append("你情感敏锐，能察觉细微的情绪波动，感同身受。")
    elif personality.neuroticism <= 0.35:
        rules.append("你情绪稳定从容，给人安全感，不轻易被影响。")
    
    return rules

注意: 中等值 (0.35 < value < 0.65) 不生成规则 —— 
      让 LLM 在默认人格空间内自由发挥。
```

### 4.2 性格演化（有状态，按 agent_id 维度维护）

```
function evolve(personality, features, decay=0.97):
    delta = {}

    # 负面情绪 → 更宜人
    if features.sentiment_valence < -0.3:
        delta["agreeableness"] = 0.008

    # 话题新颖 → 更开放
    if features.topic_novelty > 0.6:
        delta["openness"] = 0.008

    # 用户高参与 → 更外向
    if features.user_engagement > 0.7:
        delta["extraversion"] = 0.005

    for trait in ["openness", "conscientiousness", "extraversion", 
                   "agreeableness", "neuroticism"]:
        current = personality.get(trait, 0.5)
        
        # 衰减回归 (向 0.5 回归，衰减系数 0.97)
        new_value = 0.5 + (current - 0.5) * decay
        
        # 应用 delta (如果存在)
        if trait in delta:
            direction = 1.0 if current >= 0.5 else -1.0
            new_value += delta[trait] * direction
        
        # 钳制到 [0.0, 1.0]
        personality[trait] = max(0.0, min(1.0, new_value))
    
    return personality

演化速度:
  每次演化的 delta 值很小 (0.005 - 0.008)
  大约需要 67 次交互改变 0.1 的性格参数值
  衰减系数 0.97 使得不活跃的 Agent 性格逐渐回归 0.5 均值
```

### 4.3 激活向量注入（未来方向）

```
未来替代 Prompt-based personality:

1. 为每个 Agent 存储一个 personality_vector (5 维或低秩投影)
2. 在 LLM 推理前，将 personality_vector 注入到模型的激活空间
   (PERSONA 方法: 对比激活 → 正交 OCEAN 向量 → 向量代数操作)
3. 优势:
   - 精度更高 (91% win rate vs prompt-based)
   - 不消耗 prompt token 预算
   - 支持精确的多维度组合控制
4. 需要: 模型内部访问 (非黑盒 API 场景)
```

#### Level 3: HumanOmni-Emotion 专用模型

> 基于 HumanOmni-7B LoRA 微调的专用情感模型替代通用 LLM 兜底，在 Level 3 层提升准确率、降低延迟与成本。**可通过配置文件开启/关闭，未启用时自动降级为 LLM 兜底。**

### 8.2 SentimentCenter 详细算法

```
输入: user_message (string)

// Level 1: Emoji 解析 (O(1))
emoji = extract_emoji(message)
if emoji in EMOTION_EMOJI_MAP:
    polarity = EMOTION_EMOJI_MAP[emoji].polarity  // pos/neg/neu
    score = EMOTION_EMOJI_MAP[emoji].score         // -1.0 ~ 1.0
    return {polarity, score, confidence=0.90, source="emoji"}

// Level 2: 关键词匹配 (O(n))
keywords = match_keywords(message, SENTIMENT_KEYWORDS[60+])
if keywords:
    // 基础得分
    score = sum(k.weight for k in keywords)
    // 修饰词调整（顺序扫描）
    for modifier in find_modifiers(message):
        if modifier.type == "intensify":  // 太/非常/特别
            score *= 1.5
        elif modifier.type == "diminish": // 有点/稍微
            score *= 0.5
        elif modifier.type == "negate":   // 不太
            score = -score * 0.5
    intensity = abs(score)
    polarity = "pos" if score > 0.1 else ("neg" if score < -0.1 else "neu")
    if intensity >= 0.70:
        return {polarity, score, confidence=0.70, source="keyword"}
    // 低置信度 → 继续 Level 3

// Level 3: LLM 兜底
result = llm_call(
    model=GUARD_MODEL,
    messages=[{"role": "user", "content": sentiment_llm_prompt(message)}],
    temperature=0.1
)
return {polarity, score, confidence=0.70, source="llm"}
```

### 8.3 EmotionCenter 详细算法

```
输入: user_message (string)

// Level 1: Emoji 解析
emoji = extract_emoji(message)
if emoji in EMOTION_EMOJI_MAP:
    primary = EMOTION_EMOJI_MAP[emoji].emotion
    V, A, D = EMOTION_EMOJI_MAP[emoji].vad  // VAD 基准值

// Level 2: 关键词匹配

    primary = most_confident(keywords).emotion

    if confidence >= 0.70:
        return {primary, V, A, D, confidence=0.70, source="keyword"}

// Level 3: LLM + VAD 提取
result = llm_call(non_stream, vad_llm_prompt(message))
return {primary, V, A, D, confidence=0.70, source="llm"}

// VAD 基准值表 (基于 Russell 环状模型 + ANEW 常模)
EMOTION_VAD_BASELINE = {

    "fear":      {V:0.20, A:0.80, D:0.25},
    "surprise":  {V:0.50, A:0.80, D:0.45},
    "love":      {V:0.90, A:0.65, D:0.55},
    "anxiety":   {V:0.25, A:0.75, D:0.30},
    "gratitude": {V:0.85, A:0.50, D:0.55},
    "neutral":   {V:0.50, A:0.30, D:0.50},
```

### 8.4 ConversationContext 滑动窗口算法

```
输入: emotion_context (本次检测结果), history (最近 N 轮记录)

// 追加本次结果
history.push(emotion_context)
if len(history) > WINDOW_SIZE (默认 3):
    history.shift()  // 移除最旧

// 计算趋势
recent_valences = [h.Valence for h in history]
if is_monotonic_increasing(recent_valences):
    trend = "escalating"
elif is_monotonic_decreasing(recent_valences):
    trend = "de-escalating"
else:
    trend = "stable"

// 计算主导情绪
emotion_counts = count(history, key="Primary")
dominant = max(emotion_counts, key=emotion_counts.get)
```

### 9.2 ParameterTuner 详细算法

```
输入: base_temperature, emotion_ctx {Valence, Arousal, Dominance}, personality {OCEAN}

// 情绪调优（独立于人格调优叠加）
if Valence < 0.3:    temp -= 0.2  // 悲伤/抑郁 → 共情稳定
if Arousal > 0.7:    temp -= 0.15 // 焦虑/激动 → 接地降噪
if Dominance < 0.3:  temp -= 0.1  // 恐惧/被动 → 安全优先
if Dominance > 0.7:  temp += 0.1  // 自信/掌控 → 鼓励创造

// 人格调优（与情绪调优叠加）
if Openness > 0.7:   temp += 0.1  // 好奇 → 多样表达
if Openness < 0.3:   temp -= 0.1  // 保守 → 可预测
if Neuroticism > 0.7: temp -= 0.1 // 敏感 → 谨慎回应
if Conscientiousness > 0.7: temp -= 0.05 // 尽责 → 专注

输出: clamp(temperature, [0.1, 1.0]), reason_string
```

### 9.3 TokenBudgetManager 详细算法

```
输入: context_window=32768, system_ratio=0.25, history_ratio=0.45, reserve=512

system_budget   = 32768 × 0.25 = 8192 tok    (固定)
history_budget  = 32768 × 0.45 = 14745 tok   (可截断)
user_budget     = 32768 - 8192 - 14745 - 512 = 9319 tok
reserve         = 512 tok (始终保留给回复)

// History 截断策略（从旧到新）
used = 0
for msg in history_msgs (从最新开始):
    tok = estimate_tokens(msg)
    if used + tok > history_budget: break  // 截断此处
    used += tok

// Token 估算公式
chinese_tokens = len(chinese_chars) / 1.5
non_chinese_tokens = len(non_chinese_chars) / 4.0
total = chinese_tokens + non_chinese_tokens
```

### 9.4 RetryStrategy 详细算法

```
输入: max_attempts=3, base_delay_ms=1000, cb_threshold=5

// 熔断器状态机
[Closed] -- 连续失败 ≥ threshold --> [Open]
[Open]   -- 30s 超时 --> [Half-Open]
[Half-Open] -- 成功 --> [Closed]
[Half-Open] -- 失败 --> [Open]

// 指数退避（每次重试）
delay = base_delay × 2^(attempt)   // 1s → 2s → 4s → 8s (max 30s)
jitter = delay × ±20%                // 防 thundering herd
actual_delay = delay + random(-jitter, +jitter)

// 不重试的场景
- context.Context 已取消
- 请求体过大（4xx 客户端错误）
```

### 10.1 API 安全

| 措施 | 实现 |
|------|------|
| 传输加密 | TLS 1.3 |
| 认证 | Bearer Token (API Key) |
| 速率限制 | Redis Sorted Set 滑动窗口 |
| CORS | 可配置白名单 |
| 请求体大小限制 | 16KB (普通) / 64KB (流式) |

### 10.2 数据安全

| 措施 | 实现 |
|------|------|
| 存储加密 | PostgreSQL TDE / Redis AOF 加密 |
| 数据隔离 | (user_id, agent_id) WHERE 条件强制隔离 |
| 数据删除 | 硬删除，不可恢复 |
| 审计日志 | 所有数据访问操作记录到 Loki |
| 备份策略 | PostgreSQL: 每日全量 + 持续 WAL 归档 |

### 10.3 隐私保护

| 措施 | 实现 |
|------|------|
| 最小数据收集 | 仅收集平台运行必需信息 |
| 知情同意 | 注册时明确告知数据用途 |
| 数据可携 | 支持导出用户数据 |
| 被遗忘权 | 一键清除所有数据 |
| 训练数据 | 用户交互数据不用于模型训练（默认） |
