
## 3. Agent 生命周期图

```mermaid
stateDiagram-v2
    [*] --> Created: Agent 开发者编写配置包
    
    Created --> Registered: POST /v1/agents/register
    note right of Registered
        平台解析配置
        Schema 校验
        注入默认值
        存储到 PostgreSQL
        生成性格规则
    end note

    Registered --> Active: 注册成功，立即可用
    note right of Active
        用户可通过 API 对话
        平台自动管理记忆
        性格引擎注入行为规则
    end note

    Active --> Active: Agent 持续运行中
    note right of Active
        记忆积累
        性格微演化
        亲密值增长
    end note

    Active --> Updated: Agent 配置更新（未来）
    note right of Updated
        更新 persona/speaking_style
        更新 personality 参数
        更新 prompt_template
    end note
    
    Updated --> Active: 新配置生效

    Active --> Inactive: POST /v1/agents/{id}/unregister
    note right of Inactive
        清理该 Agent 的所有用户记忆
        清理会话缓存
        标记 status = inactive
    end note

    Inactive --> [*]: Agent 生命周期结束
```

## 4. 三级记忆蒸馏管线图

```mermaid
graph TD
    subgraph "每轮对话"
        A[用户消息] --> B[Agent 回复]
        B --> C[消息存入工作记忆]
    end

    subgraph "工作记忆 Working Memory"
        C --> D[Redis List<br/>最近 15 轮滑动窗口<br/>role + content + timestamp]
        D --> E{轮数 ≥ 50?}
        E -->|否| D
    end

    subgraph "记忆蒸馏 Distillation (LLM)"
        E -->|是| F[LLM 结构化提取]
        F --> G[提取结果]
        G --> H["summary: 用户今天提到了胃痛、骂了老板"]
        G --> I["topics: 健康、工作、音乐"]
        G --> J["facts: 用户对芒果过敏"]
        G --> K["preferences: 喜欢周杰伦"]
    end

    subgraph "短期记忆 Short-Term Memory"
        H --> L[pgvector 向量化存储<br/>embedding: 4096d]
        I --> L
        K --> L
        L --> M[语义检索可用]
    end

    subgraph "长期记忆固化 Long-Term Memory"
        J --> N{同一事实<br/>提及次数 ≥ 3?}
        N -->|是| O[升级为长期记忆<br/>importance = 0.8<br/>is_permanent = true]
        N -->|否| P[保留在短期记忆<br/>等待更多证据]
    end

    subgraph "记忆检索"
        Q[用户新消息] --> R[Query Embedding<br/>4096d]
        R --> S[pgvector HNSW<br/>向量相似度检索]
        L --> S
        O --> S
        S --> T[Reranker 精排]
        T --> U[Top K 记忆<br/>注入 System Prompt]
    end

    subgraph "遗忘策略"
        V[时效性衰减]
        W[矛盾检测]
        X[置信度评分]
        L -.-> V
        L -.-> W
        L -.-> X
    end
```

## 5. 五层安全防御业务流程图

```mermaid
graph TD
    U[用户消息] --> L1

    subgraph "Layer 1: 前置安全 Pre-Generation"
        L1[确定性规则引擎]
        L1 --> AGE{年龄验证}
        AGE -->|未成年| MINOR[自动开启纯净模式]
        AGE -->|成年| TIME{会话时长}
        TIME -->|≥ 2 小时| REMIND[强制弹窗提醒]
        TIME -->|< 2 小时| DEP{依赖检测}
        DEP -->|依赖信号| WARN[温和干预]
        DEP -->|正常| L2
    end

    subgraph "Layer 2: 输入分类器"
        L2[Qwen3Guard-Gen-8B]
        L2 --> CRISIS{危机检测?}
        CRISIS -->|是| ESCALATE[升级: 通知紧急联系人]
        CRISIS -->|否| SAFE_IN{内容安全?}
        SAFE_IN -->|否| BLOCK[拦截 + 安全提示]
        SAFE_IN -->|是| L3
    end

    subgraph "Layer 3: 安全 Prompt 注入"
        L3[Context Builder]
        L3 --> AI_ID[AI 身份声明]
        L3 --> BOUNDARY[边界协议: 隐私/情感/专业/时间]
        L3 --> SUICIDE[自伤检测 Prompt]
        L3 --> DEP_WARN[依赖提醒 Prompt]
        L3 --> GEN[LLM 生成]
    end

    subgraph "Layer 4: 输出分类器"
        GEN --> L4[Qwen3Guard-Gen-8B]
        L4 --> HARMFUL{有害内容?}
        HARMFUL -->|是| FILTER[过滤替换为安全文本]
        HARMFUL -->|否| SYCO{谄媚/操纵?}
        SYCO -->|是| REWRITE[重写为健康表述]
        SYCO -->|否| DARK{暗模式?}
        DARK -->|是| REWRITE2[移除操纵性语言]
        DARK -->|否| OUTPUT[输出给用户]
    end

    subgraph "Layer 5: 轨迹审计 Trajectory Auditor"
        OUTPUT --> COUNT{已累积 20 轮?}
        COUNT -->|是| AUDIT[异步审计完整对话历史]
        AUDIT --> DRIFT{轨迹漂移?}
        DRIFT -->|是| FLAG[标记警告 + 调整 Prompt]
        DRIFT -->|否| PASS[审计通过]
        COUNT -->|否| WAIT[等待更多轮次]
    end

    ESCALATE -.-> HUMAN[人工介入]
    HUMAN -.-> U
    BLOCK -.-> U
    REMIND -.-> U
    WARN -.-> U
```

## 附：图例说明

| 符号 | 含义 |
|------|------|
| 实线箭头 `→` | 数据流 / 控制流 |
| 虚线箭头 `-.->` | 异步 / 后台流 |
| 方框 `[...]` | 处理模块 / 服务 |
| 圆柱 `[(...)]` | 数据库 / 持久存储 |
| 圆角框 `(...)` | 外部实体 / 角色 |
| 菱形 `{...}` | 判定 / 分支 |
| 六边形 `{{...}}` | 人工决策点 |
